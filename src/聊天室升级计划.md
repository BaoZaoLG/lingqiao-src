# 公共聊天室升级为客户端内置“社区”页面

## Summary
将现有主界面里的小型“公共聊天”卡片升级为客户端内置的独立“社区”页面。首版保留现有轻量存储策略：**24 小时 / 最多 200 条消息 / 3 秒轮询**，重点补齐社区感和治理闭环：昵称、气泡聊天、引用回复、表情反应、在线人数、管理员删除和按“卡密+机器”禁言。

## Key Changes
- 客户端新增页面导航：主页面保留激活、注入、更新等核心功能；新增“社区”内页，激活后可进入。
- 社区页 UI：
  - 消息气泡区，区分自己、其他用户、系统消息。
  - 顶部显示“社区”、在线/最近活跃人数、连接状态。
  - 输入区支持发送、失败重试、发送中状态。
  - 支持设置昵称，默认沿用 `用户-XXXX`。
  - 支持引用回复和表情反应。
- 原主界面聊天卡片不再作为完整聊天入口，可移除或改为“社区入口 + 未读提示”。

## API / Data Changes
- 扩展 `ChatMessage`：
  - `author_id`：后端生成的稳定社区身份 ID，不暴露卡密/机器原文。
  - `author`：用户展示名，默认 `用户-XXXX`，可改昵称。
  - `reply_to_id`：可选，被引用消息 ID。
  - `reply_preview`：可选，返回被引用消息的作者、内容摘要。
  - `reactions`：表情到数量的映射，例如 `{ "👍": 3, "？": 1 }`。
  - `reacted`：当前用户已点过的表情列表。
- 新增/调整客户端 API：
  - `POST /api/v1/chat/profile`：获取/更新当前社区昵称。
  - `POST /api/v1/chat/send`：增加 `reply_to_id` 可选字段。
  - `POST /api/v1/chat/react`：对消息添加或取消表情反应。
  - `POST /api/v1/chat/presence`：更新在线状态并返回最近活跃人数。
  - `POST /api/v1/chat/messages`：继续按 `after_id` 增量拉取，并返回扩展消息字段。
- 新增管理端 API：
  - `GET /admin/api/chat/messages`：查看最近消息。
  - `POST /admin/api/chat/delete`：软删除消息。
  - `POST /admin/api/chat/mute`：按“卡密+机器”生成的社区身份禁言。
  - `POST /admin/api/chat/system`：发送系统消息。
- 存储仍使用现有 JSONStorage；消息保留策略保持 24 小时 / 200 条。

## Implementation Notes
- 后端以现有 `_src_tmp/injector-server/chat.go` 为基础扩展，不引入 SQLite 或 WebSocket。
- 客户端以现有 `_src_tmp/src/ui/main_window.h` / `impl_builder.h` / `impl_chat.h` 的 Qt 风格实现页面切换和社区页。
- 管理端在现有 admin web 中增加“社区”页，复用当前登录态、Toast、表格和 API 客户端模式。
- 禁言判断在发送消息、修改昵称、表情反应时生效；拉取消息不受禁言影响。
- 昵称限制：1-16 个字符，去首尾空白，拒绝空昵称和敏感词；展示名仅用于社区，不影响卡密/代理数据。
- 表情首版固定白名单：`👍`、`❤️`、`😂`、`？`、`收到`，避免任意文本污染 UI。
- 在线人数按最近 60 秒 presence 记录统计，不要求严格实时。

## Test Plan
- 后端单测：
  - 发送带引用消息，列表返回 `reply_preview`。
  - 昵称更新后，新消息展示新昵称，历史消息保留原展示名或按实现统一显示当前昵称。
  - 表情反应可添加/取消，同一用户同一表情不重复计数。
  - 被禁言身份无法发送、改昵称、点表情，返回明确错误。
  - 删除消息后普通列表不可见，管理端列表可识别删除状态。
  - 保留 24 小时 / 200 条裁剪逻辑不回退。
- 客户端验证：
  - 未激活时社区页显示需要激活状态。
  - 激活后可进入社区、设置昵称、发送、引用回复、点表情。
  - 断网/401/429/禁言状态有明确提示，发送失败可重试。
  - 页面切换不丢失未读计数和最后消息 ID。
- 管理端验证：
  - 管理员可查看消息、删除消息、禁言用户、发送系统消息。
  - 禁言后客户端立即在下一次操作中收到限制提示。

## Assumptions
- 社区页首版只做客户端内置 Qt 页面，不做独立 Web 用户页面。
- 消息刷新继续使用现有轮询，不引入 SSE/WebSocket。
- 消息保留继续使用 24 小时 / 200 条。
- 禁言范围为当前社区身份，即“卡密 + 机器”的稳定哈希。
- 表情反应纳入首版；更复杂的楼中楼、图片、文件、私聊不纳入首版。
